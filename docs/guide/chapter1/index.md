## 1-1 DGX Spark 是什麼

### 1-1-1 從 Project DIGITS 到 DGX Spark：產品命名演變

2025 年 1 月，NVIDIA 在 CES 展上發表了一個代號為 **Project DIGITS** 的產品。當時大家只知道這是一台「個人 AI 超級電腦」，體積很小但效能很猛。

到了 2025 年 5 月，NVIDIA 正式將它命名為 **DGX Spark**，並開始出貨。

為什麼改名？因為「DGX」是 NVIDIA 資料中心 AI 系統的知名品牌（如 DGX A100、DGX H100），把「Spark」加上去，代表這是一台「個人版」的 DGX — 小巧、Spark（火花），但同樣強大。

| 時間 | 名稱 | 狀態 |
|------|------|------|
| 2025.01 | Project DIGITS | 發表概念 |
| 2025.05 | DGX Spark | 正式命名、開始出貨 |

### 1-1-2 產品定位：誰需要 DGX Spark？

簡單來說，DGX Spark 是給 **「想在自家書桌上跑 AI 模型」** 的人設計的。

具體來說：

- **AI 開發者**：需要在本地測試大模型，不想每次都花錢租雲端 GPU
- **研究人員**：需要微調模型、做實驗，但預算買不起 H100 伺服器
- **創作者**：想用 AI 生成圖片、影片、音樂，但不想依賴雲端服務
- **企業 IT**：需要在本地部署 AI 應用，資料不能離開公司

::: tip 💡 一句話總結
DGX Spark = 一台放在書桌上、有 128GB 記憶體、能跑 120B 大模型的個人 AI 超級電腦。
:::

### 1-1-3 DGX Spark 在 NVIDIA 產品線中的位置

NVIDIA 的 AI 硬體產品線可以這樣理解：

```
資料中心等級
  ├─ DGX H200    → 141 GB HBM3e，數十萬美元，機房等級
  ├─ DGX B200    → 最新 Blackwell 架構，百萬美元等級
  │
個人工作站等級
  ├─ DGX Spark   → 128 GB LPDDR5x，約 $3,999，書桌等級 ← 你在這裡
  │
消費級顯示卡
  ├─ RTX 5090    → 32 GB GDDR7，約 $2,000
  ├─ RTX 4090    → 24 GB GDDR6X
```

DGX Spark 的獨特之處在於：**它的 128GB 記憶體比 RTX 5090 的 32GB 大了整整 4 倍**。這意味著你能裝進記憶體的模型大了 4 倍。

### 1-1-4 DGX Spark 的 OEM 生態系

NVIDIA 不只自己賣 DGX Spark，還授權給其他廠商生產相容設備，包括：

- **ASUS（華碩）**
- **GIGABYTE（技嘉）**
- **MSI（微星）**
- **Supermicro（美超微）**

這些 OEM 廠商推出的產品核心規格相同，但可能在機殼設計、散熱、保固服務上有所差異。

## 1-2 GB10 Grace Blackwell 超級晶片

DGX Spark 的核心是一顆叫做 **GB10** 的超級晶片。這個名字來自兩個架構的結合：

- **Grace** = NVIDIA 的 ARM CPU 架構（以物理學家 Grace Hopper 命名）
- **Blackwell** = NVIDIA 的最新 GPU 架構（以數學家 David Blackwell 命名）

### 1-2-1 晶片架構：NVIDIA × MediaTek 聯合設計

GB10 不是 NVIDIA 一個人做出來的。它結合了：

- **NVIDIA**：負責 GPU 部分、NVLink 互連、軟體堆疊
- **MediaTek（聯發科）**：負責 CPU 部分的設計與製造

這種合作模式讓 GB10 能在保持高效能的同時，控制成本和功耗。

### 1-2-2 CPU：20 核心 ARM 處理器

DGX Spark 的 CPU 是一顆 **20 核心的 ARM 處理器**（具體來說是 ARM Neoverse V2 核心）。

| 規格 | 數值 |
|------|------|
| 核心數 | 20 核 |
| 架構 | ARM Neoverse V2 |
| 線程數 | 20 線程（無超執行緒） |

::: info 🤔 ARM 跟我們平常用的 x86 有什麼不同？
你平常用的 Intel/AMD 處理器是 x86 架構，而 DGX Spark 用的是 ARM 架構（跟手機晶片同家族）。ARM 的好處是 **省電、發熱少**，壞處是 **有些軟體可能沒有 ARM 版本**。

不過別擔心，本書用到的所有工具（Ollama、Docker、Python 等）都有 ARM 版本。
:::

### 1-2-3 GPU：Blackwell 架構

GPU 部分是 NVIDIA 最新的 **Blackwell 架構**。這是繼 Ampere（RTX 30 系列）、Ada Lovelace（RTX 40 系列）之後的新一代架構。

Blackwell 的重點特色：
- 更強的 AI 運算能力
- 原生支援 FP4 精度（後面會詳細解釋）
- 更好的記憶體頻寬效率

### 1-2-4 第 5 代 Tensor Core：FP4 原生運算的威力

Tensor Core 是 NVIDIA GPU 中專門做 AI 運算的硬體單元。DGX Spark 搭載的是 **第 5 代 Tensor Core**，它最大的亮點是：

**原生支援 FP4（4-bit 浮點數）運算！**

什麼叫 FP4？簡單來說：

| 精度 | 位元數 | 能表示的數字範圍 | 用途 |
|------|--------|------------------|------|
| FP32 | 32-bit | 非常精確 | 科學計算 |
| FP16 | 16-bit | 夠用 | 一般 AI 推論 |
| FP8  | 8-bit  | 勉強夠 | 高效能推論 |
| **FP4** | **4-bit** | **有限但夠用** | **超大模型推論** |

FP4 的好處是：**同樣大小的記憶體，可以裝下 8 倍大的模型**。

::: tip 💡 為什麼這很重要？
用 FP16 跑一個 120B 模型需要約 240GB 記憶體，DGX Spark 的 128GB 裝不下。
但用 FP4 跑，只需要約 60GB，輕鬆裝進 128GB！

這就是為什麼 DGX Spark 能跑 120B 模型的秘訣。
:::

### 1-2-5 NVLink-C2C：CPU 與 GPU 之間的超高速互連

傳統電腦的 CPU 和 GPU 是透過 PCIe 匯流排溝通的，速度有限。

DGX Spark 用的是 **NVLink-C2C（Chip-to-Chip）**，這是 NVIDIA 自己設計的超高速互連技術：

- **頻寬**：900 GB/s（比 PCIe 5.0 x16 快了約 5 倍）
- **延遲**：極低
- **好處**：CPU 和 GPU 可以像使用同一塊記憶體一樣工作

這正是「統一記憶體」能實現的關鍵技術。

### 1-2-6 RT Cores（光線追蹤核心）

DGX Spark 的 GPU 也包含 **RT Cores**，也就是用來做光線追蹤運算的硬體單元。

雖然這本書主要講 AI 應用，但 RT Cores 在以下場景也有幫助：
- 3D 渲染
- 神經輻射場（NeRF）訓練
- 某些視覺 AI 模型

### 1-2-7 統一記憶體：128 GB LPDDR5x

這是 DGX Spark **最重要**的規格，沒有之一。

**傳統架構**（一般電腦）：
```
CPU ←→ 系統記憶體（例如 32GB DDR5）
  │
  └─ PCIe 匯流排（速度慢）
        │
        GPU ←→ 顯示記憶體（例如 24GB GDDR6X）
```

**DGX Spark 架構**：
```
CPU ←→ NVLink-C2C ←→ GPU
          │
          └─ 統一記憶體（128GB LPDDR5x）
```

在傳統架構中，CPU 和 GPU 各有自己的記憶體，資料要傳來傳去。在 DGX Spark 中，**CPU 和 GPU 共享同一塊 128GB 記憶體**。

這意味著：
- GPU 可以直接存取全部 128GB
- 不需要在 CPU 和 GPU 之間複製資料
- 模型載入速度更快、效率更高

### 1-2-8 運算能力：1 PFLOP FP4

DGX Spark 的 AI 運算能力達到 **1 PFLOP（每秒一千兆次浮點運算）**，這是基於 FP4 精度的數字。

PFLOP 是什麼？就是 Peta-FLOP = 10^15 FLOP = 每秒 1,000,000,000,000,000 次運算。

作為比較：
- 一台 MacBook Pro M3 Max：約 0.1 PFLOP
- 一張 RTX 4090：約 1.3 PFLOP（但只有 24GB 記憶體）
- **DGX Spark：1 PFLOP + 128GB 記憶體** ← 記憶體才是關鍵

### 1-2-9 與資料中心 Blackwell 的差異

雖然都叫 Blackwell，但 DGX Spark 的 GB10 和資料中心的 B200 是不同的晶片：

| 特性 | GB10（DGX Spark） | B200（資料中心） |
|------|-------------------|------------------|
| 記憶體 | 128 GB LPDDR5x | 192 GB HBM3e |
| 記憶體頻寬 | ~273 GB/s | ~8 TB/s |
| 功耗 | ~150W | ~1000W |
| 價格 | ~$3,999 | ~$30,000+ |
| 用途 | 個人 AI 工作站 | 資料中心叢集 |

DGX Spark 的記憶體頻寬比較低，但對於推論來說已經足夠。

## 1-3 128 GB 統一記憶體：改變遊戲規則的關鍵

### 1-3-1 為什麼記憶體比算力更重要？

在 AI 領域有一句名言：

> 「算力決定速度，記憶體決定能不能跑。」

意思是：如果你的模型太大、裝不進記憶體，那算力再強也沒用。

舉個例子：
- 一個 120B 參數的模型，用 FP16 精度需要約 **240GB** 記憶體
- 用 FP4 精度需要約 **60GB** 記憶體
- DGX Spark 有 **128GB** → 用 FP4 可以輕鬆跑，還能留空間給其他程式

### 1-3-2 統一記憶體的獨特優勢

統一記憶體不只是「記憶體很大」這麼簡單，它還帶來了以下好處：

1. **零拷貝**：CPU 和 GPU 之間不需要複製資料
2. **動態分頁**：如果記憶體真的不夠，系統可以自動把不常用的資料移到 SSD
3. **簡化程式設計**：開發者不需要手動管理 CPU/GPU 記憶體

### 1-3-3 128 GB 能做什麼？實戰場景一覽

| 應用 | 所需記憶體 | DGX Spark 能做到嗎？ |
|------|-----------|---------------------|
| 跑 8B 語言模型 | ~16 GB | ✅ 輕鬆 |
| 跑 70B 語言模型 | ~35-70 GB | ✅ 沒問題 |
| 跑 120B 語言模型 | ~60 GB | ✅ 可以！ |
| 微調 8B 模型（LoRA） | ~30-50 GB | ✅ 可以！ |
| 微調 FLUX 圖片模型 | ~40-60 GB | ✅ 可以！ |
| 跑 405B 語言模型 | ~200 GB | ❌ 裝不下 |
| 預訓練 1B 模型 | ~20-40 GB | ✅ 可以！ |

### 1-3-4 記憶體頻寬與推論速度的關係

記憶體頻寬決定了模型推論的速度上限。公式很簡單：

```
推論速度 ≈ 記憶體頻寬 / 模型大小
```

DGX Spark 的記憶體頻寬約為 **273 GB/s**。以一個 FP4 量化的 60GB 模型來說：

```
理論最大速度 ≈ 273 / 60 ≈ 4.5 次推論/秒
```

實際上會因為其他因素（KV cache、批次大小等）有所不同，但這給你一個基本概念。

### 1-3-5 與 Apple M3 Ultra、AMD Ryzen AI Max+ 395 的比較

| 設備 | 統一記憶體 | 記憶體頻寬 | AI 算力 | 價格 |
|------|-----------|-----------|---------|------|
| **DGX Spark** | **128 GB** | **273 GB/s** | **1 PFLOP** | **~$3,999** |
| Mac Studio M3 Ultra | 192 GB | 800 GB/s | ~0.4 PFLOP | ~$6,000+ |
| AMD Ryzen AI Max+ 395 | 128 GB | ~200 GB/s | ~0.3 PFLOP | ~$1,500+ |

DGX Spark 的優勢在於 **專屬的 NVIDIA 軟體生態系**（CUDA、TensorRT、NGC 等），這在 AI 領域是無可取代的。

## 1-4 機身與連接埠

### 1-4-1 外觀尺寸與重量

DGX Spark 非常小巧：

- **尺寸**：約 15 × 15 × 5 公分（比一個便當盒還小）
- **重量**：約 1.2 公斤
- **顏色**：NVIDIA 經典的綠色 + 黑色

你可以輕鬆把它放在書桌上，甚至塞進背包裡帶著走。

### 1-4-2 後方面板連接埠

DGX Spark 的後方提供了豐富的連接埠：

| 連接埠 | 數量 | 用途 |
|--------|------|------|
| HDMI 2.1 | 1 | 連接顯示器 |
| USB-A 3.2 | 2 | 鍵盤、滑鼠、隨身碟 |
| USB-C | 1 | 資料傳輸 |
| 200GbE 網路 | 1 | 高速網路（ConnectX-7） |
| 電源 | 1 | DC 電源輸入 |

### 1-4-3 無線連線

DGX Spark 內建 **Wi-Fi 6E** 和 **藍牙 5.3**，可以：
- 無線連線到家中網路
- 連接藍牙鍵盤、滑鼠、耳機

### 1-4-4 高速網路：ConnectX-7 200GbE

這是 DGX Spark 最誇張的規格之一 — 它內建了 **NVIDIA ConnectX-7 200GbE 網路卡**。

200GbE 是什麼概念？
- 一般家用網路：1 GbE
- 一般企業網路：10 GbE
- **DGX Spark：200 GbE**

這意味著兩台 DGX Spark 之間可以直接用網路線連接，進行高速資料傳輸，為後續的多機互連（第 25 章）做準備。

::: info 🤔 我家沒有 200GbE 的網路設備怎麼辦？
別擔心，DGX Spark 也支援一般的 1GbE 網路（透過 USB 轉接或 Wi-Fi）。200GbE 主要是用於多機互連的場景。
:::

### 1-4-5 儲存

DGX Spark 內建 **1TB NVMe SSD**。

1TB 夠用嗎？來看看模型的大小：

| 模型 | 大小 |
|------|------|
| 8B 模型（FP4） | ~4 GB |
| 70B 模型（FP4） | ~35 GB |
| 120B 模型（FP4） | ~60 GB |
| FLUX.1-dev 圖片模型 | ~23 GB |

所以 1TB 可以裝下很多模型。如果不夠，還可以透過 USB 外接 SSD 擴充。

### 1-4-6 電源與散熱設計

- **電源**：外部電源供應器，約 150W 最大功耗
- **散熱**：被動散熱 + 風扇，運轉時非常安靜

DGX Spark 的功耗只有約 150W，比一般桌上型電腦（300-500W）省電很多。

## 1-5 開箱前的準備清單

::: warning ⚠️ 重要！
在收到 DGX Spark 之前，請先準備好以下物品。否則開箱後會發現沒東西可以接！
:::

### 1-5-1 必備：HDMI Dummy Plug（假負載插頭）

::: tip 💡 這是什麼？為什麼需要？
HDMI Dummy Plug 是一個小小的插頭，插在 HDMI 孔上，讓系統以為有接顯示器。

為什麼需要？因為 DGX Spark 設定完成後會改為「Headless 模式」（不接顯示器，遠端操控），但如果完全沒接顯示器，有些桌面環境會無法正常啟動。

**價格**：約 $3-5 美元，網路上到處都買得到。
:::

### 1-5-2 必備：首次開機設定用的周邊

首次開機時，你需要：
- **HDMI 顯示器**（或電視）
- **USB 鍵盤**
- **USB 滑鼠**

設定完成後就可以拔掉，改用遠端連線。

### 1-5-3 必備：網路設備

- **網路線**（Cat 6 或以上，建議 Cat 6a）
- **路由器 / 交換器**

### 1-5-4 必備：另一台電腦

你需要另一台電腦（筆電或桌機都行）來：
- SSH 遠端連線到 DGX Spark
- 下載模型和工具
- 日常開發

### 1-5-5 選配：雙機互連用線材

如果你計劃未來做多機互連（第 25 章），可以準備：
- **DAC 線（Direct Attach Cable）**：用於兩台 DGX Spark 直連
- **200GbE 交換器**：用於三台以上互連
